# SKY-UX-017 — Three-AI Consensus (new way)

**Date:** 2026-07-19  
**Brief:** Apply Responsive Priority System to ShortKey homepage — protect L1, solve mobile competition.  
**Sifu:** Sky observes · Family teaches with Reason · Approach · Why · Expect  
**Top 3 (UX weights):** Sonnet **40%** · Fable **25%** · GPT **20%**

---

## 1. Sonnet — UX lead (40%)

**Reason:** Sticky signup strip + dual CTAs + tall poster fight Logo/Headline/Primary CTA on &lt;768px.  
**Approach:** Classify strip as L2; collapse to seal + one action on mobile; hero = Logo → H1 → Primary CTA first; secondary CTA full-width under; cap poster height.  
**Why:** Priority system — L1 never loses to L2/L3. Thumb reach + clarity.  
**Expect:** Mobile first screen reads Logo, headline, Start Beauty Match without hunting.

## 2. Fable — Brand (25%)

**Reason:** Premium minimalism dies when chrome stacks. S+ is family, not clutter.  
**Approach:** Keep Dark Luxury; monochrome logo untouched; lilac only on primary buttons; hide lockup copy on xs; no new badges.  
**Why:** Simplex-ity pause + ShortKey DNA — subtract.  
**Expect:** Hero still feels expensive, not busy.

## 3. GPT — Conversion (20%)

**Reason:** Two equal CTAs + two strip buttons dilute the money path (Beauty Match → shop).  
**Approach:** One primary path above the fold; Marketplace = L2; strip = supporting revenue, not rival to hero.  
**Why:** Decision filter — revenue + understanding.  
**Expect:** Clear conversion spine without removing signup entirely.

---

## Sky consensus (weighted)

| Move | Level | Action |
|------|-------|--------|

| Headline + Start Beauty Match | L1 | Protect; primary first on mobile |

**FinalScore spine:** Sonnet structure + Fable restraint + GPT conversion path.

**Solution (MVP — Sifu):** Implement mobile collapse on strip + hero CTA order + poster height cap. Manual log to Sky M. No over-engineering.
